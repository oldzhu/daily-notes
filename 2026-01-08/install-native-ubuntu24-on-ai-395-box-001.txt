=======
You’ve uncovered the key detail: your earlier “C vs D” mapping by serial was flipped. The new Get-Partition output is the authoritative mapping.

From your Get-Partition table:

Disk 0, partition 4 = DriveLetter C (≈ 1.999 TB)
→ Windows system disk is Disk 0
→ Serial ends with 4998 is C

Disk 1, partition 2 = DriveLetter D (≈ 2.000 TB)
→ Data disk is Disk 1
→ Serial ends with A451 is D

So:

C-disk serial: …4998
D-disk serial: …A451
That’s what you must use in the Ubuntu installer.

What to do now (safe dual-boot on D)
When you boot the Ubuntu live USB, run:

sudo nvme list
Then:

The NVMe whose serial ends with A451 is your D disk → OK to wipe/install Ubuntu there.
The NVMe whose serial ends with 4998 is your C disk (Windows) → do NOT touch it.
Why the confusion happened
Windows drive letters (C/D) are about partitions/volumes. Disk numbers are just OS enumeration. The reliable “ground truth” is always partition → drive letter, which you printed with Get-Partition.

One more safety guard (recommended)
Before installing Ubuntu, in Windows Disk Management:

Right‑click D: → Delete Volume (optional but makes it very obvious in the Ubuntu installer that the target disk is “empty/unallocated”).
Not required, but it reduces risk.
Installation choice
Use the “Something else” method and select the disk by serial A451. Create:

EFI 1GB FAT32 /boot/efi
/ ext4 300–400GB
home ext4 rest of disk
